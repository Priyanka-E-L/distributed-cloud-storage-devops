✓
B1 — Add Docker's official GPG key and repository
We install Docker Engine directly (not Docker Desktop). This is the correct method for Ubuntu on WSL2.
UBUNTU
# Add Docker's GPG key
install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
  | gpg --dearmor -o /etc/apt/keyrings/docker.gpg
chmod a+r /etc/apt/keyrings/docker.gpg

# Add Docker's stable apt repository
echo \
  "deb [arch=$(dpkg --print-architecture) \
  signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" \
  | tee /etc/apt/sources.list.d/docker.list > /dev/null

apt update

⚠ COMMON MISTAKE — typo in hello-world
docker run hello world ← WRONG (space instead of hyphen)
Always use a hyphen: docker run hello-world
With a space, Docker reads "hello" as the image and "world" as a command — causing "repository not found".

from flask import Flask, jsonify
import redis, os

app = Flask(__name__)

r = redis.Redis(
    host=os.getenv("REDIS_HOST", "localhost"),
    port=6379,
    decode_responses=True
)

@app.route("/")
def index():
    count = r.incr("visits")
    return jsonify({
        "message": "Hello from the DevOps Lab!",
        "hostname": os.uname().nodename,
        "visits": count
    })

@app.route("/health")
def health():
    return jsonify({"status": "ok"})

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=5000, debug=True)

Write the Dockerfile

✓
F1 — Create the Dockerfile and .dockerignore
The Dockerfile is a recipe for building a portable, repeatable image of your app. The .dockerignore stops unnecessary files from being copied in.
UBUNTU
nano ~/devops-lab/app/Dockerfile
# Paste the content below, then Ctrl+O → Enter → Ctrl+X
app/Dockerfile
# Base image — slim Python runtime
FROM python:3.12-slim

# Set working directory inside the container
WORKDIR /app

# Copy requirements FIRST for layer caching
COPY requirements.txt .

# Install Python packages
RUN pip install --no-cache-dir -r requirements.txt

# Copy the rest of the app code
COPY . .

# Environment variables
ENV FLASK_ENV=production
ENV REDIS_HOST=redis

# Port the app listens on
EXPOSE 5000

# Health check
HEALTHCHECK --interval=30s --timeout=5s \
  CMD curl -f http://localhost:5000/health || exit 1

# Start app with gunicorn
CMD ["gunicorn", "--bind", "0.0.0.0:5000", "--workers", "2", "app:app"]
UBUNTU — create .dockerignore
nano ~/devops-lab/app/.dockerignore
# Paste below, then Ctrl+O → Enter → Ctrl+X
app/.dockerignore
venv/
__pycache__/
*.pyc
.env
*.md
ℹWhy copy requirements.txt before the code? Docker caches each layer. If only your code changes (not requirements), Docker reuses the cached pip install layer — saving minutes on every rebuild.

G1 — Create Nginx config and Dockerfile
Nginx forwards all incoming requests to the Flask app using the Docker Compose service name as a hostname.
UBUNTU
nano ~/devops-lab/nginx/nginx.conf
# Paste below, then Ctrl+O → Enter → Ctrl+X
nginx/nginx.conf
events {}

http {
    upstream flask_app {
        server app:5000;   # "app" is the Docker Compose service name
    }

    server {
        listen 80;

        location / {
            proxy_pass       http://flask_app;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }
    }
}

G2 — Create docker-compose.yml
This file defines all three services and wires them together on a private Docker bridge network.
UBUNTU
nano ~/devops-lab/docker-compose.yml
# Paste below, then Ctrl+O → Enter → Ctrl+X
docker-compose.yml
services:

  redis:
    image: redis:7-alpine
    volumes:
      - redis_data:/data
    networks:
      - backend
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  app:
    build: ./app
    image: devops-lab:v1
    environment:
      - REDIS_HOST=redis
    networks:
      - backend
    depends_on:
      redis:
        condition: service_healthy
    restart: unless-stopped

  nginx:
    build: ./nginx
    ports:
      - "80:80"
    networks:
      - backend
    depends_on:
      - app
    restart: unless-stopped

volumes:
  redis_data:

networks:
  backend:
    driver: bridge
